# $nextTick

vm.$nextTick(callback)

# 用法

在下次 DOM 更新循环结束之后执行延迟回调。在修改数据之后使用它，然后等待 DOM 更新。

2.1.0 ，如果没有提供回调且在支持 promise 的环境中，则返回一个 promise。

# 原理

Vue 在更新 DOM 时是异步执行的。只要侦听到数据变化，Vue 会开启一个事件队列，并缓冲在同一事件循环中发生的所有数据变更。如果同一个 watcher 被多次触发，只会被推入到事件循环队列中一次。在缓冲时去除重复数据对于避免不必要的计算和 DOM 操作很有用的。然后，在下一个事件循环 tick 中，Vue 刷新事件队列并执行已经去重的工作。

在调用 Watcher 的更新函数时，并不会直接进行更新，而是把需要更新的 Watcher 加入到 Queue 队列里，然后把具体的更新方法 flushSchedulerQueue 传给 nextTick 调用。

主要是因为 DOM Tree 的更新是实时的，并不需要监听到 DOM 更新，但是多次操作 DOM 的代价太过于昂贵。

这样对于性能来说无疑是好的，但是也带来了问题，DOM 异步更新，修改数据之后，去获取 DOM 节点内容，是获取到的之前的数据，这个时候还没有更新 DOM。如果想要基于 DOM 更新后做一些事情，可以使用 $nextTick 来做。

# 触发视图更新

Vue 在调用 Watcher 更新视图的时候，不会直接进行更新，而是把需要更新的 Watcher 加入到 Queue 队列中，然后把具体更新方法 flushSchedulerQueue 传给 nextTick 调用。

// src/core/observer/watcher.js 
class Watcher {
    update () {
        ...
        queueWatcher(this)
    }
}

特殊的情况，deep、lazy 。 Watcher 的两种用法，一种是更新模版、另一种是监听某个值的变化。

  1. 模版更新 在 Vue 生命周期挂载元素时，通过创建 Watcher 对象，然后 updateComponent 来更新渲染模版。

vm._watcher = new Watcher(vm, updateComponent, noop);

创建 Watcher 会调用 this.get , 也就是 updateComponent. 在 render 的过程中，会调用 data 的 getter 方法, 以此来建立数据的双向绑定, 当数据改变时，会重新出发 updateComponent. this.get 返回值是 undefined，主要是用来渲染模版。

  1. 监听数据 另一个用途就是 computed、watch 等，监听数据的变化来执行相应的操作。

this.get 返回的就是要监听数据的值。初始化过程中，调用 this.get 会拿到初始化值保存为 this.value, 监听的数据改变之后，会再次调用 this.get 并拿到修改之后的值，将旧值和新值一起传给 cb 并执行回调。

# queueWatcher

把当前的 watcher 添加到一个队列中。Vue 中页面的更新是异步的，所以一系列数据的变化，会在之后某一时刻统一更新。

// src/core/observer/scheduler.js
function queueWatcher(watcher) {
    // watcher 有从小到大的唯一 id，在页面更新时，会按照一定的顺序依此更新。
    const id = watcher.id
    // 如果没有存过 watcher 的 id
    if (has[id] == null) {
        // 存下来
        has[id] = true
        // 如果不是处于刷新队列状态
        if (!flushing) {
            // 将当前 Watcher 添加到异步队列。
            queue.push(watcher);
        }

        // 如果没有处在正在刷新，就去刷新队列
        if (!waiting) {
            waiting = true;
            // 在下一个 nextTick 中执行 flushSchedulerQueue
            nextTick(flushSchedulerQueue)
        }
    }
}

// 更新视图的具体方法
function flushSchedulerQueue() {
    // 排序，先渲染父节点，再渲染子节点
    queue.sort((a, b) => a.id - b.id);

}

# nextTick

// 回调队列
const callbacks = []
// 异步锁
let pending = false

// 执行队列中的每一个回调
function flushCallbacks () {
  pending = false  // 重置异步锁
  // 防止出现 nextTick 中包含 nextTick 时出现的问题，在执行回调函数队列之前，提前
  // 复制并清空回调函数队列
  const copies = callbacks.slice(0)
  callbacks.length = 0
  // 执行回调函数队列
  for (let i = 0; i < copies.length; i++) {
    copies[i]()
  }
}

function nextTick (cb) {
    // 将回调函数推入回调队列。
    callbacks.push(() => {
        if (cb) {
            try {
                cb.call(ctx)
            } catch (e) {
                handleError(e, ctx, 'nextTick')
            }
        } else if (_resolve) {
            _resolve(ctx)
        }
    })

    // 如果异步锁未锁上，锁上异步锁，调用异步函数，等同步函数执行完之后，就开始执行回调函数队列。
    if (!pending) {
        pending = trues
        // 执行， 会在下一个 tick 执行 执行 flushCallbacks,
        // 就是对 callbacks 遍历，然后执行相对应的回调函数。
        timerFunc()
    }
    
}
  1. 把传入的回调函数 cb 压入 callbacks 数组
  2. 执行 timerFunc 函数，延迟调用 flushCallbacks 函数
  3. 遍历执行 callbacks 数组中的所有函数。

调用 this.$nextTick 其实就是调用 nextTick 方法，在异步队列中执行回调函数。

根据先进先出的原则，修改 data 触发的更新异步队列先得到执行，执行完之后就生成新的 DOM。
